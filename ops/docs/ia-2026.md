# IA 2026 вЂ” page-by-page disposition table (working doc)

**What this is:** the proposed content consolidation behind the five-door IA

**How to read the table:**

| Disposition | Meaning |
|---|---|
| **Keep** | Stays as a page at its URL (possibly retitled/moved in nav) |
| **Refresh** | Keep, but the content needs a writing pass (noted why) |
| **Merge в†’ X** | Content folds into page X; old URL 301s there |
| **Directory entry** | Becomes an entry on the /engage ministry directory (name, blurb, when/where, contact); old URL 301s to its directory anchor |
| **Redirect в†’ X** | Unpublish + 301 (no content worth carrying) |
| **Archive** | Unpublish, no redirect (nothing links to it; recoverable from trash/DB) |
| **Owner call** | Needs a human decision before touching |

**Ground rules:** unpublish, never delete (everything is recoverable);
every removed URL that ever appeared in a bulletin/e-news gets a 301
(Redirection plugin / MCP redirect abilities); ministry pages have owners вЂ”
**validate the "Directory entry" rows with the office before executing.**

**Net effect:** ~70 published pages в†’ ~40, no orphan sections, every nav
door lands on substance.
